## ❓ שאלות - displayInfo.height

```
□ יש input validation ל-height בקליינט?
□ מוגבל רק לערכים חיוביים?
□ יש min/max מוגדרים?
□ איזה סוג input? (text, number, slider)
```

**דוגמת הבעיה:**
```json
{"displayInfo": {"height": -100}}  // ⚠️ BE מקבל!
```

## ❓ שאלות - nfftSelection

```
□ יש dropdown עם [256, 512, 1024, 2048] בלבד?
□ או input חופשי?
□ יש ווולידציה power-of-2?
□ יש הגבלה max=2048?
```

**ערכים תקינים:**
```
256, 512, 1024, 2048  // רק אלה!
```

**דוגמת הבעיה:**
```json
{"nfftSelection": 1000}  // ⚠️ לא power-of-2, BE מקבל!
{"nfftSelection": 4096}  // ⚠️ מעל 2048, BE מקבל!
```

## ❓ שאלות - frequencyRange (קריטי!)

```
□ הקליינט יודע מה ה-PRR של ה-dataset?
□ הקליינט מחשב Nyquist limit (PRR/2)?
□ הקליינט מגביל את המשתמש לפי Nyquist?
□ איך הקליינט מקבל את ה-PRR? (metadata API?)
□ מה קורה אם dataset משתנה?
```

**הבעיה המורכבת:**
- Nyquist הוא **דינמי** (תלוי ב-dataset)
- דוגמה: PRR=1000 Hz → Nyquist=500 Hz
- אם משתמש מבקש max=600 → **שגוי!**

**דוגמת הבעיה:**
```json
{
  "frequencyRange": {"min": 0, "max": 600}
  // ⚠️ אם PRR=1000, Nyquist=500, אז 600 שגוי!
}
```

## ❓ שאלות - channels

```
□ יש ווולידציה של max-min+1 <= 2500?
□ איך המשתמש בוחר? (slider, multi-select, range?)
□ מה קורה אם מנסה לבחור >2500?
□ הקליינט מחשב את הכמות בזמן אמת?
```

**החלטה מהפגישה:**
- מקסימום **2500 ערוצים** לכל טאב
- חישוב: `channel_count = max - min + 1`

**דוגמת הבעיה:**
```json
{"channels": {"min": 1, "max": 2501}}  // ⚠️ 2501 ערוצים, BE מקבל!
```
