## פירוט לפי עדיפות

### 🔴 עדיפות קריטית (חייב לקבל)

#### 1. Kubernetes API Access
**מדוע קריטי:**
- ללא זה לא ניתן לפתח בדיקות מתקדמות
- מגביל את היכולת לניטור בזמן אמת
- נדרש לכל בדיקות Kubernetes

**מה נדרש:**
- kubeconfig file עם credentials תקפים
- גישה ישירה ל-API Server
- או: VPN access ל-internal network

**תלות:** אין

**זמן יישום:** 1-2 שבועות

**השפעה:** גבוהה מאוד - מאפשר פיתוח בדיקות מתקדמות

---

#### 2. ServiceAccount + RBAC
**מדוע קריטי:**
- נדרש ל-CI/CD integration
- מאפשר הרצת בדיקות אוטומטיות
- נדרש לניטור resources

**מה נדרש:**
- ServiceAccount: `focus-automation-sa`
- Role עם הרשאות מינימליות
- RoleBinding

**תלות:** #1 (Kubernetes API Access)

**זמן יישום:** 3-5 ימים

**השפעה:** גבוהה מאוד - מאפשר CI/CD integration

---

#### 8. Network/VPN Access
**מדוע קריטי:**
- נדרש לגישה ישירה מ-CI/CD runners
- מאפשר פיתוח יעיל יותר
- נדרש לבדיקות מתקדמות

**מה נדרש:**
- VPN access ל-internal network
- או: Network routing מתאים
- או: LoadBalancer IPs עם firewall rules

**תלות:** אין

**זמן יישום:** 2-3 שבועות

**השפעה:** גבוהה מאוד - מאפשר פיתוח יעיל

---

### 🟡 עדיפות חשובה (מומלץ לקבל)

#### 3. MongoDB Advanced Permissions
**מדוע חשוב:**
- נדרש לבדיקות recovery
- נדרש לבדיקות data integrity
- מאפשר בדיקות backup/restore

**מה נדרש:**
- User עם הרשאות `backup` + `restore`
- גישה ל-`admin` database

**תלות:** אין

**זמן יישום:** 2-3 ימים

**השפעה:** בינונית - משפר יכולות בדיקה

---

#### 4. סביבת Dev/Test נפרדת
**מדוע חשוב:**
- מאפשר פיתוח בטוח
- מאפשר בדיקות הרסניות ללא סיכון
- מאפשר ניסויים טכניים

**מה נדרש:**
- Namespace נפרד: `focus-automation-dev`
- MongoDB/RabbitMQ/Focus Server נפרדים

**תלות:** #1, #2

**זמן יישום:** 2-4 שבועות

**השפעה:** גבוהה - מאפשר פיתוח בטוח

---

#### 5. Metrics & Monitoring Access
**מדוע חשוב:**

- מאפשר זיהוי resource exhaustion

**מה נדרש:**
- גישה ל-Prometheus (אם קיים)
- גישה ל-Kubernetes metrics API
- גישה ל-Grafana dashboards

**תלות:** #1

**זמן יישום:** 1-2 שבועות

**השפעה:** בינונית - משפר יכולות ניטור

---

#### 9. CI/CD Secrets Management
**מדוע חשוב:**
- נדרש ל-CI/CD integration
- מאפשר אבטחה טובה יותר
- מאפשר אוטומציה מלאה

**מה נדרש:**
- GitHub Secrets מוגדרים
- Kubernetes credentials
- Database credentials
- SSH keys

**תלות:** #1, #2

**זמן יישום:** 3-5 ימים

**השפעה:** בינונית - משפר CI/CD

---

#### 10. Logs Centralized Access

**תלות:** #1

**זמן יישום:** 1 שבוע
