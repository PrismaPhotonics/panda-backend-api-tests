## מורכבות המערכת והאתגרים

### 1. מורכבות טכנית

#### רכיבי המערכת

**12 שירותי ליבה:**
1. Supervisor - מנהל תהליכים (514 שורות קוד)
2. Preprocessor - עיבוד אותות (C++ executable)
3. Peripherals - ממשק חומרה
4. Smart Recorder - הקלטת PRP
5. Heatmap Recorders - 10+ סוגי Heatmaps
6. Baby Analyzer - עיבוד ביניים
7. Fiber Inspector - ניטור סיב (OTDRX/Y)
8. Data Manager - ניהול דיסק
9. BIT - בדיקות מובנות
10. Telegraf - איסוף מטריקות
11. RabbitMQ - Message Broker
12. MongoDB - מסד נתונים

**תשתיות:**
- RabbitMQ עם 10+ תורים
- MongoDB עם מספר collections
- NAS/Storage עם FIFO rollover
- Kubernetes (בחלק מהפריסות)
- Control Centre integration
- Focus Server integration

#### חקירות היסטוריות - מגבלות עומס

**התנהגות עומס:**
- עומס בחקירות היסטוריות יקרה רק כשיהיו הרבה חקירות פעילות במקביל
- חקירה נחשבת פעילה כל עוד היא מעבדת דאטא
- קצב עיבוד: פי ~200 מזמן אמת (הערכה, לא מאומת)

**מגבלות פרק זמן:**
- מגבלה על פרק זמן לפי גודל החלון: עד **30 חלונות**
- חישוב: `משך זמן מקסימלי = גודל_חלון × 30`

**דוגמאות:**
- אם גודל החלון הוא **30 שניות** → מקסימום **15 דקות** (30 × 30 = 900 שניות)
- אם גודל החלון הוא **יום** → מקסימום **30 יום** (1 × 30 = 30 ימים)

**השלכות לבדיקות:**
- פרק זמן ארוך יותר מאפשר להכריח שהחקירה תרוץ יותר זמן
- לבדיקות עומס: צריך ליצור חקירות עם פרקי זמן ארוכים כדי להבטיח שהן יישארו פעילות
- לבדיקות יציבות: צריך לבדוק התנהגות עם מספר חקירות פעילות במקביל

#### אתגרים טכניים

| אתגר | מורכבות | השפעה |
|------|---------|--------|
| **זרימת נתונים מורכבת** | 🔴 גבוהה | Digitizer → Preprocessor → ml_algo/smart_recorder/baby_analyzer |
| **תלויות בין שירותים** | 🔴 גבוהה | Supervisor מנהל תלויות, race conditions |
| **תצורות מורכבות** | 🔴 גבוהה | 5 ורטיקלים × 4 סוגי פריסה × תצורות HW |
| **תהליכי אינטגרציה** | 🟡 בינונית-גבוהה | תהליך שחרור גרסה מורכב |
| **Debugging** | 🔴 גבוהה | לוגים מרובים, מערכות מבוזרות |
| **תרחישי שגיאה** | 🔴 גבוהה | Network failures, disk full, NAS down, etc. |
